Overview

As UX Researcher, I led user interviews, usability testing, and creator workflow analysis to evaluate Opus Clip's AI-assisted editing experience — synthesizing findings into a low-fidelity mockup for a redesigned mobile workflow.

Problem

Content creators needed a way to use AI editing anywhere — but Opus Clip required them to upload video from a cloud drive first, rather than directly from their device.

Creators had to complete several setup steps before Opus Clip could generate clips. The product lacked a mobile version, required local file uploads instead of cloud-based imports, and asked users to manually provide editing details such as subtitle style, video length, and video topic. These requirements added friction to the AI-assisted editing workflow.

Solution

Develop a mobile app with an open upload portal, so creators can upload directly from local storage instead of a cloud drive first — with AI auto-detection deciding editing details on their behalf.

The redesign expands upload options to support both local and cloud-based sources, and uses AI auto-detection to reduce manual setup. Instead of requiring users to enter subtitle style, video length, and topic manually, the system suggests smart defaults and lets creators adjust only when needed.
